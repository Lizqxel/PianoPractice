# Chord Sprint

コードネームを見て素早く鍵盤を押さえるための、MIDIキーボード対応ピアノコード練習Webアプリです。バックエンドやアカウントは使わず、履歴はブラウザの `localStorage` に保存します。

公開版: **https://lizqxel.github.io/PianoPractice/**

## 起動方法

Node.js 20以降を推奨します。

```bash
npm install
npm run dev
```

ターミナルに表示されたURL（通常は `http://localhost:5173`）をChromeまたはEdgeで開いてください。

品質チェック:

```bash
npm test
npm run build
```

## MIDIキーボードの接続

1. MIDIキーボードをUSBでPCへ接続し、電源を入れます。
2. ChromeまたはEdgeでChord Sprintを開きます。
3. 画面右上の「MIDI機器を接続」を押します。
4. ブラウザのMIDIアクセス確認を許可します。
5. 複数の入力機器がある場合は、右上の一覧から使用する機器を選びます。

MIDI機器がなくても、画面下のC2〜C6仮想鍵盤をマウスまたはタッチで操作できます。MIDI入力と仮想鍵盤のどちらでも内蔵音源が鳴り、同じコード判定が動作します。

## Studio One／外部DAW音源

音源設定を「Studio One／外部DAW」にすると、受信したMIDIを選択したMIDI出力へ転送します。内蔵鍵盤音は停止しますが、コード判定、画面上の鍵盤、練習記録、内蔵メトロノームはそのまま利用できます。「内蔵＋外部の両方」も選択できます。

Windowsでは仮想MIDIポートを作成し、次のように接続します。

1. 「Chord Sprint Out」という仮想MIDIポートを作成します。
2. Chord Sprintの入力に実物のMIDIキーボード、出力にChord Sprint Outを選びます。
3. Studio One 6の外部デバイスでNew Keyboardを追加し、Receive FromをChord Sprint Out、Send ToをNoneにします。
4. Instrument Trackを作成し、好きな音源とプリセットを選んでモニターをONにします。

ブラウザからStudio One内のVSTやプリセットを取得することはできません。音源とプリセットの選択はStudio One側で行います。MIDI入出力に同じポートを選ぶとループする可能性があるため、画面の警告が出た場合は設定を見直してください。

## 収録機能

- コード瞬発: 今日・白鍵・メジャー・マイナー・苦手・カスタム・全コードから出題範囲を選択
- コード別成績: 出題数・正解率・平均反応を保存し、苦手コードを自動抽出
- コード進行: 4種類の進行、全12キー、40〜160 BPM、Web Audio基準の同期メトロノームと4カウント
- 60秒コードチェンジ: 共通音を残した滑らかなコード移動に対応
- 14日間プラン: Day別の問題・合格条件、自動記録・自動完了、CSV出力
- 初心者向けガイド学習: 形を覚える → 2秒ヒント → 自力確認の3フェーズ
- Day別専用レッスン: 転回形、左右手、進行、キー変更、コード譜、初見、総合テスト
- 仮想鍵盤ガイド: 紫の押す鍵盤、緑の正解、赤の余分、左手水色、指番号表示
- 左右手分析: 分割点を設定し、右手転回形と左手ベースを個別表示
- 分数コード: C/E、C/G、D/F#、G/Bなどの右手コードとベースを個別判定
- リアルタイムコード検出: major、minor、dim、sus4、7、maj7、m7、add9と転回形
- Web Audio音源: sine + triangle、16音ポリフォニック、音量・ミュート
- 外部DAW音源: Note On/Off、Velocity、チャンネル、CC、サステイン、Pitch Bend、AftertouchをMIDI出力へ転送
- メトロノーム: アクセント、音量、4カウント
- 集中モード

## 14日間プランの流れ

- Day 1〜3: ガイドを見ながらコードの形を覚え、最後にガイドなしで2回正解する
- Day 4〜5: 転回形と左手ベース＋右手コード
- Day 6・10: メトロノーム同期のコード進行とキー変更
- Day 7・13: コード瞬発テストと苦手コード集中
- Day 8〜9: 4音コードと分数コード
- Day 11: 自分で入力できるコード譜エディタ
- Day 12: 10秒の確認時間から始まる初見コード譜
- Day 14: 瞬発、転回形、分数、進行、初見の総合テスト

コード別にガイドあり／なし正解数、ミス、ヒント使用、反応時間、最終練習日時、習得状態をブラウザへ保存します。

## 対応ブラウザ

- Google Chrome（最新版）
- Microsoft Edge（最新版）

Web MIDI APIを利用するため、FirefoxとSafariは対象外です。MIDI権限は `localhost` またはHTTPSの安全なコンテキストでのみ利用してください。

## トラブルシューティング

### MIDI機器が見つからない

- キーボードの電源とUSB接続を確認し、ページを再読み込みします。
- DAWなど、MIDI機器を排他的に使用するアプリを終了します。
- USBハブを外し、PC本体のUSBポートへ直接接続します。
- ブラウザのサイト設定でMIDI権限を一度リセットし、再接続します。

### 音が鳴らない

- 画面を一度クリックしてから鍵盤を弾きます。ブラウザはユーザー操作前の音声開始を制限します。
- 左下の音源がミュートになっていないか、音量が0でないか確認します。
- OSの出力先と音量を確認します。

### 音が鳴り続ける

ウィンドウからフォーカスが外れると自動で全音停止します。改善しない場合はページを再読み込みしてください。

### 履歴が保存されない

プライベートブラウズや、サイトデータを終了時に削除する設定では `localStorage` が消える場合があります。通常ウィンドウで使用してください。

## GitHub Pagesへのデプロイ

`main` ブランチへpushすると、[Deploy to GitHub Pages](.github/workflows/deploy-pages.yml) が自動で次を実行します。

1. `npm ci`
2. `npm test`
3. `npm run build`
4. `dist` をGitHub Pagesへデプロイ

Viteの `base` はリポジトリ名に合わせて `/PianoPractice/` に設定されています。手動で再実行する場合は、GitHubの **Actions → Deploy to GitHub Pages → Run workflow** を使用してください。
